# 人才获取

人才获取是一个普遍话题，这里是把这个普遍话题放在C++开发的上下文中来讨论。

我们需要“有心有力”的人才。“有心”是指正心、积极心、持久心、探究心。“有力”是指专业能力、学习能力、解难（实际问题）能力。
为了得到真正的、合适的人才，应该在面试中摒弃“面试八股文”，而是根据面试者的个人特点、能力特长以及工作经历来探寻他的“心与力”。基于这个考量，面试可以分为两个部分：

1. “有力”
如何解决实际工作中的问题，涉及到哪些专业知识，有没有快速学习新知识，解决问题的逻辑是否清晰，是否有章法（往大里说是“方法论”），是否涉及到沟通协调的问题，如何把沟通协调变顺畅的。

例如：性能优化相关的计算机基础包括：算法分析、内存体系、多核编程、CPU体系、Cache友好、并发与异步等。可以借此考察面试者对基础理论的理解，注意不要考背书，而是考理解和应用。

2. “有心”
为什么选择这个职位，对未来的规划，可稍微涉及到人生目标。借此机会考察面试者的三观。看他/她是“长期主义者”还是“投机主义者”；是真正热爱这项工作还是为了养家糊口（养家糊口也没错）；是风险厌恶的还是风险偏好的；是抗压还是不抗压，等等。
